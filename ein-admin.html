Save as `admin.html` 2. Put `logo2.png` in same folder 3. Replace Firebase configuration 4. Deploy securely 5. Protect admin page with Firebase Authentication before production use --- ## Firebase Realtime Database Rules Use these Firebase Realtime Database rules while testing: ```json { "rules": { "einApplications": { ".read": true, ".write": true } } } ``` For production, secure the database using Firebase Authentication. --- ## Firebase Hosting Commands ```bash npm install -g firebase-tools firebase login firebase init firebase deploy ``` --- ## Folder Structure ```text project-folder/ │ ├── index.html ├── admin.html ├── logo2.png └── firebase.json ``` --- ## Important Both pages use the same Firebase path: ```text einApplications ``` So submissions from the public page automatically appear inside the admin dashboard in realtime. --- ## Optional Future Upgrades * Admin login authentication * Approve / Reject buttons * Export to PDF * Export to Excel * Email notifications * SMS notifications * Application tracking numbers * Applicant portal login * Status update system * Dark mode dashboard
